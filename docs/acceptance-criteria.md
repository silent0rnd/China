# Acceptance Criteria

## 1. Source of truth

- [ ] Прочитаны `PROJECT_STATE.md`, `strategy-spec.md`, `page-spec.md`, `design-spec.md`, `technical-spec.md` и `README.md`.
- [ ] Тексты, порядок блоков, CTA, поля и факты не изменены.
- [ ] Контрольные изображения используются только как visual reference.
- [ ] Неподтверждённые данные из изображений не перенесены.

## 2. Визуальное соответствие

- [ ] Сайт не выглядит как типовой AI-шаблон.
- [ ] Сохранена концепция «Тёмный логистический коридор».
- [ ] Нет белых секций, glassmorphism, случайных градиентов и абстрактного 3D.
- [ ] Красный остаётся точечным и не доминирует.
- [ ] Фирменный маршрутный принцип заметен без анимации.
- [ ] Композиция, типографика, реальные фотографии и CTA соответствуют утверждённому впечатлению.
- [ ] Визуальное качество не упрощено ради более лёгкой реализации.
- [ ] Реальные фотографии не заменены стоками или генерациями.
- [ ] Подписи к фото не содержат вымышленных моделей, маршрутов, сроков, веса или результатов.

## 3. Hero

- [ ] `product-design:image-to-code` использован для desktop/mobile hero, если Skill доступен.
- [ ] Desktop 1440 x 900 совпадает с прототипом по крупным зонам, пропорциям и иерархии.
- [ ] Mobile 390 x 844 сохраняет порядок header -> eyebrow -> H1 -> subtitle -> CTA -> facts -> scene.
- [ ] H1 не пересекается со сценой.
- [ ] Основной CTA виден без прокрутки на 390 x 844.
- [ ] Hero остаётся понятным при отключённой анимации.
- [ ] После hero выполнен `web-design-guidelines` и устранены критические замечания.

## 4. Структура страницы

- [ ] Все обязательные 12 блоков `page-spec.md` реализованы.
- [ ] Архитектура «доказательства -> процесс -> расчёт» сохранена.
- [ ] Процесс содержит шесть этапов.
- [ ] Сроки отмечены как ориентировочные.
- [ ] Тариф от 70 ₽/кг не стал главным оффером.
- [ ] Официальная и карго-схема показаны нейтрально.
- [ ] Нет обещаний фиксированной цены или доставки любого груза за 5 дней.

## 5. Responsive QA

Проверить ширины:

- [ ] 320 px;
- [ ] 360 px;
- [ ] 390 px;
- [ ] 430 px;
- [ ] 768 px;
- [ ] 1024 px;
- [ ] 1280 px;
- [ ] 1440 px;
- [ ] 1920 px.

Проверить высоты:

- [ ] 667 px;
- [ ] 740 px;
- [ ] 800 px;
- [ ] 900 px;
- [ ] 1080 px.

Дополнительно:

- [ ] Нет горизонтального scroll страницы на 320 px.
- [ ] Mobile не является простым уменьшением desktop.
- [ ] Галерея mobile прокручивается вручную и показывает часть следующего кадра.
- [ ] Процесс mobile вертикальный.
- [ ] Sticky CTA не перекрывает форму и системные элементы.
- [ ] Touch targets не меньше 44 x 44 px.

## 6. Формы

### Основная

- [ ] Form ID `cargo_calculation`.
- [ ] Два этапа.
- [ ] File upload доступен на первом этапе.
- [ ] Обязательны только тип груза, точки забора и доставки, имя, контакт и consent.
- [ ] Frontend validation совпадает с server-side правилами.
- [ ] Ошибки находятся рядом с полями.
- [ ] Loading блокирует повторную отправку.
- [ ] Назад не очищает введённые данные.
- [ ] Success полностью заменяет форму.
- [ ] Success показывается только после `ok: true` сервера.
- [ ] `503 integration_not_configured` не показывается как success.

### Короткая

- [ ] Form ID `logist_request`.
- [ ] Состав полей совпадает с `page-spec.md`.
- [ ] Используется тот же server endpoint и consent logic.

### Files

- [ ] Разрешены JPG, JPEG, PNG, PDF, DOC, DOCX, XLS, XLSX.
- [ ] Максимум один файл и 15 MB.
- [ ] MIME и расширение проверяются на сервере.
- [ ] Временный файл удаляется после обработки.
- [ ] Исполняемые форматы отклоняются.

### Anti-spam

- [ ] Honeypot работает.
- [ ] Submission token проверяется.
- [ ] Idempotency предотвращает дубль.
- [ ] CAPTCHA не добавлена без утверждения.

## 7. Analytics

- [ ] Counter ID задаётся только после утверждения.
- [ ] Script не загружается до analytics consent.
- [ ] `trackGoal` не ломает сайт при blocked analytics.
- [ ] `form_success_*` вызывается только после server success.
- [ ] Двойной click не дублирует success.
- [ ] В событиях и network requests нет PII.
- [ ] UTM first-touch соответствует `analytics-plan.md`.
- [ ] Development logs не содержат значения формы.

## 8. Cookie

- [ ] Banner содержит три утверждённые кнопки.
- [ ] Analytics выключена до выбора.
- [ ] Reject сохраняет выбор и не блокирует сайт.
- [ ] Footer повторно открывает настройки.
- [ ] Сохраняются version, categories и timestamp.
- [ ] Consent banner не перекрывает CTA и поля.

## 9. Legal pages

- [ ] Доступны `/privacy-policy/`, `/personal-data-consent/`, `/cookie-policy/`.
- [ ] Все документы имеют `Draft for legal review` до проверки.
- [ ] Checkbox не отмечен по умолчанию.
- [ ] Links consent кликабельны.
- [ ] Consent формы отделён от cookie.
- [ ] Рекламная рассылка не включена.
- [ ] Неиспользуемые сервисы не упомянуты как активные.
- [ ] Перед production заполнены юридический и почтовый адреса.

## 10. Accessibility

- [ ] Контраст текста соответствует утверждённым значениям.
- [ ] Visible focus есть у всех интерактивных элементов.
- [ ] Цвет не единственный индикатор.
- [ ] Menu, modal, lightbox и accordion работают с клавиатуры.
- [ ] Focus возвращается инициатору.
- [ ] Form errors связаны через `aria-describedby`.
- [ ] Success/error объявляются live region.
- [ ] Alt описывает только видимое содержание.
- [ ] Zoom не заблокирован.
- [ ] `prefers-reduced-motion` сохраняет полностью рабочую страницу.

## 11. Performance

- [ ] Выполнены бюджеты `technical-spec.md`.
- [ ] Hero имеет отдельные desktop/mobile assets.
- [ ] Некритичные изображения lazy.
- [ ] Критичные размеры медиа заданы.
- [ ] Не более четырёх WOFF2-файлов.
- [ ] Нет лишних runtime-библиотек.
- [ ] Нет console errors и unhandled promise rejections.
- [ ] LCP, INP и CLS фактически измерены после публикации.
- [ ] Lighthouse-оценка не заявлена до измерения.

## 12. Security

- [ ] Secrets отсутствуют в Git и frontend.
- [ ] `config.local.php` исключён из Git.
- [ ] Production `display_errors` выключен.
- [ ] Служебные PHP-файлы недоступны по HTTP.
- [ ] Server validation обязательна.
- [ ] CORS не открыт на `*`.
- [ ] External blank links используют noopener.
- [ ] Dependency audit выполнен.
- [ ] Server logs не содержат body формы и PII.

## 13. SEO

- [ ] Title, description и H1 совпадают с `page-spec.md`.
- [ ] Один H1.
- [ ] Canonical содержит утверждённый production-домен.
- [ ] `robots.txt` проверен.
- [ ] `sitemap.xml` содержит production URL.
- [ ] Open Graph настроен.
- [ ] Favicon работает.
- [ ] HTTPS работает.
- [ ] 404 возвращает корректный status при поддержке хостинга.
- [ ] Redirect map утверждён до замены старого сайта.
- [ ] Нет фиктивных canonical и host names.

## 14. Build and tests

- [ ] `npm ci` проходит.
- [ ] `composer install --working-dir=server` проходит.
- [ ] `npm run lint` проходит.
- [ ] `npm run test` проходит.
- [ ] `npm run build` проходит.
- [ ] `npm run test:e2e` проходит.
- [ ] Production build не содержит source secrets.
- [ ] Preview соответствует build, а не dev server.

## 15. Codex QA

- [ ] Использованы и перечислены фактические Skills.
- [ ] `web-design-guidelines` выполнен после hero и перед final.
- [ ] `product-design:audit` выполнен перед приёмкой.
- [ ] Расхождения с прототипами закрыты или явно согласованы.
- [ ] Итоговый отчёт содержит build, tests, QA widths, open issues и changed files.

## 16. Production blockers

До снятия всех пунктов сайт не считается готовым к публикации:

- [ ] Утверждён домен.
- [ ] Утверждён hosting и document root.
- [ ] Настроен counter ID.
- [ ] Настроен основной получатель.
- [ ] Настроен резервный канал или явно утверждено его отсутствие.
- [ ] Проверена test-заявка с вложением.
- [ ] Заполнены юридический и почтовый адреса.
- [ ] Юридические документы проверены.
- [ ] Создан backup текущего сайта.
- [ ] Утверждён rollback plan.
